# UI Review Checklist — Telier

## 1. Estrutura
- A unidade principal exibida é tarefa ou projeto (não decoração)?
- A hierarquia da tela está clara em 1 leitura?

## 2. Metadados
- Metadados estão em formato label + valor?
- Não há pills/badges/chips como estrutura principal?

## 3. Componentes
- Sidebar, topbar, headers, cards e listas seguem padrão existente?
- Não há variações visuais conflitantes do mesmo componente?

## 4. Layout
- A interface está compacta e escaneável?
- Não há espaço desperdiçado entre blocos operacionais?

## 5. Métricas
- Métricas simples estão em metric strip?
- Foram evitados cards grandes para informação curta?

## 6. Tipografia
- Papéis de título, label, valor e descrição estão consistentes?
- Não existem estilos tipográficos conflitantes na mesma tela?

## 7. Cor
- Cor está sendo usada como sinal de estado/foco?
- Não existe uso decorativo excessivo de cor?

## 8. Responsividade
- Mobile/tablet funcionam sem quebrar fluxo?
- Não há overflow horizontal em headers, strips e listas?

## 9. Consistência
- A tela parece parte do mesmo sistema visual do Telier?
- Estados de hover/ativo/foco estão discretos e previsíveis?

## 10. Regressões
- Algo voltou ao estilo legado (pills, badges, cards soltos)?
- Há duplicação de informação que deveria ser consolidada?
